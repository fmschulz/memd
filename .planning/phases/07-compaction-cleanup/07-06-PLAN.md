  <action>
Create `evals/harness/src/suites/compaction.rs` following existing suite patterns:

1. Test case enum:
   ```rust
   #[derive(Debug, Clone)]
   pub enum CompactionTest {
       F1TombstoneFiltering,      // Deleted chunks never in search results
       F2SegmentMerge,            // Segment count reduces after merge
       F3HnswRebuild,             // HNSW staleness reduces after rebuild
       F4ResultsInvariant,        // Search results same chunk IDs before/after (minus deleted)
       F5LatencyDuringCompaction, // Measure p50/p99 while compaction runs
       F6ForceCompaction,         // Force flag works regardless of thresholds
   }
   ```

2. Test implementations:

   **F1TombstoneFiltering:**
   - Add 10 chunks, delete 5
   - Search for content from deleted chunks
   - Verify none of the deleted chunk IDs appear in results

   **F2SegmentMerge:**
   - Check memory.stats for segment_count
   - Run memory.compact with force=true
   - Verify segment_count in stats decreased (or stayed same if already minimal)

   **F3HnswRebuild:**
   - Add chunks, delete some, check hnsw_staleness in stats
   - Run memory.compact
   - Verify hnsw_staleness decreased

   **F4ResultsInvariant:**
   - Add 20 chunks with known content
   - Delete 5 chunks
   - Search for query that matches non-deleted chunks
   - Record result chunk IDs (as a SET - order may vary due to HNSW rebuild)
   - Run memory.compact
   - Search again with same query
   - Verify result chunk IDs match (same SET of IDs)

   NOTE: We verify same IDs (as a set), NOT strict ordering. HNSW rebuild may produce
   slightly different scores due to graph structure changes, which can affect ordering.
   The invariant is: "same chunks are returned" not "same exact order".

   **F5LatencyDuringCompaction:**
   - Add many chunks (100+) to ensure compaction takes time
   - Start compaction (force=true)
   - While compaction runs, measure search latency
   - Record p50, p99 latency
   - Verify p99 < 500ms (configurable threshold)
   - Note: This is best-effort since compaction may complete quickly

   **F6ForceCompaction:**
   - With minimal data (below thresholds)
   - Run memory.compact with force=false, verify skipped
   - Run memory.compact with force=true, verify completed

3. Quality thresholds:
   - F1: 100% (no deleted chunks in results)
   - F2: segment_count_after <= segment_count_before
   - F3: hnsw_staleness_after <= hnsw_staleness_before
   - F4: 100% (same chunk IDs as set)
   - F5: p99 < 500ms during compaction
   - F6: force flag behavior correct

4. Add `pub mod compaction;` to suites/mod.rs

       println!("\n=== Suite F: Compaction ===\n");

       let suite = CompactionSuite::new(client.clone());
       let results = suite.run_all().await?;

       // Print results
       for result in &results {
           let status = if result.passed { "PASS" } else { "FAIL" };
           println!("[{}] {} - {}", status, result.test_name, result.details);
       }

       let passed = results.iter().filter(|r| r.passed).count();
       let total = results.len();
       println!("\nCompaction Suite: {}/{} passed", passed, total);

       if passed < total {
           std::process::exit(1);
       }

       Ok(())
   }
