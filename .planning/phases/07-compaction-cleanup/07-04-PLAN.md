     1. Record start time, log "compaction started" with tenant_id

        - Log merge result

     7. Cache Invalidation:
        - Call semantic_cache.invalidate_chunks(&deleted_chunk_ids)
        - Log entries invalidated

     8. Calculate total duration, log "compaction complete"

     9. Return CompactionResult

```rust
/// Get all deleted chunk IDs for a tenant
pub fn get_deleted_chunk_ids(&self, tenant_id: &TenantId) -> Result<Vec<ChunkId>> {
    let conn = self.conn.lock().unwrap();

    let mut stmt = conn.prepare(
        "SELECT chunk_id FROM chunks WHERE tenant_id = ?1 AND status = 'deleted'"
    )?;

    let rows = stmt.query_map(rusqlite::params![tenant_id.as_str()], |row| {
        let chunk_id_str: String = row.get(0)?;
        Ok(chunk_id_str)
    })?;

    let mut chunk_ids = Vec::new();
    for row in rows {
        let chunk_id_str = row?;
        if let Ok(chunk_id) = ChunkId::parse(&chunk_id_str) {
            chunk_ids.push(chunk_id);
        }
    }

    Ok(chunk_ids)
}
```

  <action>
Add methods to `DenseSearcher`:

1. `fn get_deleted_internal_ids(&self, tenant_id: &TenantId, deleted_chunk_ids: &[ChunkId]) -> HashSet<usize>`
   Logic:
   - Get tenant's HNSW index
   - For each deleted_chunk_id, look up internal_id in mapping
   - Return set of internal IDs

   Note: This requires access to the index mapping. May need to add a method to HnswIndex.

2. `fn rebuild_hnsw_for_tenant(&self, tenant_id: &TenantId, deleted_chunk_ids: &[ChunkId]) -> Result<RebuildResult>`
   Logic:
   - Get or create tenant index
   - Get deleted internal IDs
   - Call HnswRebuilder::rebuild_clean()
   - Create new HnswIndex with the rebuilt Hnsw
   - Swap the index atomically (replace in tenant_indices map)
   - Return RebuildResult

The key insight: DenseSearcher already has per-tenant indices in a RwLock<HashMap>. We can rebuild in background and then swap the entry under write lock. Queries using the old index will complete, new queries will use new index.

Alternative simpler approach: Add `rebuild_clean_in_place(&self, deleted_chunk_ids: &HashSet<ChunkId>)` to HnswIndex itself:
- Lock mapping and cache
- Create new Hnsw from cache, filtering deleted
- Swap Hnsw under write lock

This keeps the rebuild logic closer to the index.
