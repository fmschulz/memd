Create `crates/memd/src/compaction/hnsw_rebuild.rs` with:

1. `RebuildResult` struct:
   - `embeddings_processed: usize` - total embeddings in cache
   - `embeddings_included: usize` - embeddings in new index (after filtering)
   - `embeddings_excluded: usize` - deleted embeddings skipped
   - `duration: std::time::Duration`
   - Derive Debug, Clone

2. `HnswRebuilder` struct:
   - No fields (stateless utility)

3. Implement `HnswRebuilder`:
   - `fn new() -> Self`

   - `fn rebuild_clean(
        &self,
        source_index: &HnswIndex,
        deleted_internal_ids: &HashSet<usize>,
        config: &HnswConfig,
     ) -> Result<(Hnsw<'static, f32, DistCosine>, RebuildResult)>`

     Logic:
     - Record start time
     - Create new Hnsw with same config parameters
     - Iterate source_index.embedding_cache via `iter_valid()`
     - For each (internal_id, embedding):
       - If internal_id NOT in deleted_internal_ids:
         - Insert into new Hnsw
         - Increment embeddings_included
       - Else:
         - Increment embeddings_excluded
     - Calculate duration
     - Return (new_hnsw, RebuildResult)

   NOTE: This returns a raw Hnsw, not HnswIndex. The caller (CompactionManager) is responsible for creating a new HnswIndex and swapping it atomically. This separation allows the rebuild to be done in background while the old index serves queries.

4. Add `pub mod hnsw_rebuild;` to compaction/mod.rs

Imports:

- `use anndists::dist::distances::DistCosine;`
- `use hnsw_rs::hnsw::Hnsw;`
- `use std::collections::HashSet;`
- `use std::time::Instant;`

1. `MergeResult` struct:
   - `segments_before: usize`
   - `segments_after: usize`
   - `segments_merged: usize`
   - `docs_before: u64`
   - `docs_after: u64`
   - `duration: std::time::Duration`
   - Derive Debug, Clone

2. `SegmentMerger` struct:
   - `min_segments_for_merge: usize` (default 4)
   - `max_docs_before_merge: usize` (default 100_000)
   - `del_docs_ratio: f32` (default 0.2 = 20%)

3. Implement `SegmentMerger`:
   - `fn new() -> Self` with defaults
   - `fn with_config(min_segments: usize, max_docs: usize, del_ratio: f32) -> Self`

   - `fn merge(&self, index: &Bm25Index) -> Result<MergeResult>`

     Logic:
     - Record start time
     - Get segments_before from index (Tantivy provides this via reader.searcher().segment_readers().len())
     - Get docs_before from index.total_docs()
     - Configure and apply LogMergePolicy:
       ```rust
       // Tantivy's IndexWriter handles merging automatically on commit
       // Force a commit to trigger any pending merges
       index.commit()?;

       // The merge policy runs during commit, so just record new stats
       ```
     - Note: Tantivy's LogMergePolicy is set during index creation. Since our Bm25Index already creates an IndexWriter, we can trigger merges by calling commit(). More aggressive merging would require recreating the writer with a custom policy, which is deferred.

     - Get segments_after and docs_after
     - Calculate duration
     - Return MergeResult

   - `fn needs_merge(&self, current_segment_count: usize) -> bool`
     Returns true if current_segment_count > min_segments_for_merge

4. Add `pub mod segment_merge;` to compaction/mod.rs
