**structural/parser.rs - LanguageSupport:**

Enum SupportedLanguage:
- Rust, Python, TypeScript, JavaScript, Go, Java, Cpp

Struct LanguageSupport:
- language: tree_sitter::Language
- parser: tree_sitter::Parser (wrapped in RefCell for mutation)

Impl LanguageSupport:
- for_extension(ext: &str) -> Option(Self)
  Match extensions:
  - "rs" -> Rust (tree_sitter_rust::LANGUAGE)
  - "py" -> Python (tree_sitter_python::LANGUAGE)
  - "ts", "tsx" -> TypeScript (tree_sitter_typescript::LANGUAGE_TYPESCRIPT or TSX)
  - "js", "jsx" -> JavaScript (tree_sitter_javascript::LANGUAGE)
  - "go" -> Go (tree_sitter_go::LANGUAGE)
  - "java" -> Java (tree_sitter_java::LANGUAGE)
  - "cpp", "cc", "cxx", "c", "h", "hpp" -> Cpp (tree_sitter_cpp::LANGUAGE)
  - _ -> None

- for_language(lang: SupportedLanguage) -> Self

- parse(&mut self, source: &str) -> Option(tree_sitter::Tree)
  Use parser.parse(source, None)

- parse_with_old_tree(&mut self, source: &str, old_tree: &tree_sitter::Tree) -> Option(tree_sitter::Tree)
  For incremental parsing: parser.parse(source, Some(old_tree))

Struct ParseResult:
- tree: tree_sitter::Tree
- language: SupportedLanguage
- source_bytes: usize

Function detect_language(path: &Path) -> Option(SupportedLanguage):
- Extract extension from path
- Map to SupportedLanguage enum

Function parse_file(path: &Path, content: &str) -> Result(ParseResult):
- Detect language from path extension
- Get LanguageSupport for language
- Parse content
- Return ParseResult with tree and metadata

Thread safety: Parser is not Send/Sync, so create per-thread or wrap in appropriate synchronization. For now, create fresh LanguageSupport per parse (cheap operation).
