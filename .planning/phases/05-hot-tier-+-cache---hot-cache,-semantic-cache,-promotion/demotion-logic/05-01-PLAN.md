  <action>
Create cache module with AccessTracker for tracking chunk access patterns.

1. Create `crates/memd/src/cache/mod.rs`:
```rust
//! Hot tier caching layer
//!
//! Provides fast access to frequently-used chunks via:
//! - HotTier: Separate HNSW index optimized for small datasets
//! - AccessTracker: LFU-based access tracking with multi-signal scoring
//! - SemanticCache: Query embedding cache (added in 05-02)

mod access_tracker;
mod hot_tier;

pub use access_tracker::{AccessStats, AccessTracker, PromotionConfig, PromotionScore};
pub use hot_tier::{HotTier, HotTierConfig};
```

2. Create `crates/memd/src/cache/access_tracker.rs`:

Key structures:
- AccessStats: frequency (u32), last_access_ms (i64), project_id (Option<String>), manually_hot (bool)
- PromotionConfig: freq_weight, recency_weight, project_boost, manual_boost, recency_decay_ms, promotion_threshold (default 5-7 accesses)
- PromotionScore: compute() method combining signals with log-scale frequency

Key methods:
- new(config) -> Create tracker with moka::sync::Cache for stats
- record_access(chunk_id, project_id) -> Increment counter, update timestamp, store project
- mark_hot(chunk_id) -> Set manually_hot flag
- get_score(chunk_id, active_project) -> Compute promotion score with project match
- should_promote(chunk_id, active_project) -> Returns true if score exceeds threshold
- remove(chunk_id) -> Remove tracking for demoted chunks

Use moka::sync::Cache with:
- max_capacity: 100_000 (track up to 100K chunks)
- time_to_idle: 10 minutes (auto-cleanup of cold entries)

Promotion score formula (from RESEARCH.md):
```rust
let freq_score = (frequency as f32).ln_1p();  // Log scale
let recency_score = 1.0 / (1.0 + (age_ms as f32 / config.recency_decay_ms));
let project_boost = if matches_active_project { config.project_boost } else { 0.0 };
let manual_boost = if manually_hot { config.manual_boost } else { 0.0 };
config.freq_weight * freq_score + config.recency_weight * recency_score + project_boost + manual_boost
```

Default config values (from CONTEXT.md):
- promotion_threshold: 6 (middle of 5-7 range)
- freq_weight: 0.4
- recency_weight: 0.3
- project_boost: 0.2
- manual_boost: 0.3
- recency_decay_ms: 300_000 (5 minutes)

Key methods:

- search(query_embedding, k) -> Search hot tier only, returns SearchResult vec
- contains(chunk_id) -> Check if chunk is in hot tier

- should_evict(chunk_id, now_ms) -> True if past min_residency and not recently accessed
- get_eviction_candidates(now_ms, count) -> Get chunks eligible for eviction (LFU order)
- remove(chunk_id) -> Remove from hot tier (eviction)
- record_access(chunk_id, project_id) -> Delegate to AccessTracker
- get_promotion_score(chunk_id, active_project) -> Delegate to AccessTracker

HNSW config for hot tier (from RESEARCH.md - optimized for small index):

```rust
HnswConfig {

    max_elements: 2_000,      // Per CONTEXT.md target
    dimension,
}
```
