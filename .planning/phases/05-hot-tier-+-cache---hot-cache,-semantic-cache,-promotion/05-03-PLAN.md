1. Start timing
2. Cache lookup (if enabled):
   - cache.lookup(query_embedding, tenant_id, project_id, warm_tier.get_version())
   - If hit: record timing, return results with source_tier=Cache
   - Increment cache stats

3. Hot tier search (if enabled and cache miss):
   - hot_tier.search(query_embedding, k)
   - If results found: merge with warm tier results
   - Record timing

4. Warm tier search (always as fallback):
   - warm_tier.search(query_embedding, k)
   - Record timing

5. Merge and deduplicate results:
   - Prefer hot tier scores over warm tier
   - Combine both result sets, deduplicate by chunk_id
   - Sort by score descending, take top k

6. Record access for all returned chunks:
   - access_tracker.record_access(AccessEvent { chunk_id, timestamp, project_id, query_id })

7. Cache results (if cache enabled):
   - cache.insert(query_embedding, tenant_id, project_id, results, version)

8. Return TieredSearchResult with timing and source info

fn check_promotions(&self, project_id: Option<&str>) -> Vec(TierDecision):
1. Get top candidates from access_tracker.get_top_candidates(k=100, project_id)
2. Filter to chunks NOT already in hot tier
3. Filter to chunks with score >= promotion_threshold
4. For each candidate:
   - Get embedding from warm_tier.get_embedding(chunk_id)
   - If embedding exists: hot_tier.promote(chunk_id, embedding, tenant_id)
   - Record TierDecision with reason
5. Return list of decisions

fn check_demotions(&self) -> Vec(TierDecision):
1. Increment query_counter
2. If query_counter < demotion_queries_threshold: return empty
3. Reset query_counter
4. Get all chunks in hot tier
5. For each chunk:
   - Get promotion_score from access_tracker
   - If score < (promotion_threshold * 0.5): demote
   - Record TierDecision with reason
6. Return list of decisions

fn run_maintenance(&self):
1. check_promotions(None) -- no project context
2. check_demotions()
3. hot_tier.evict_if_needed(warm_tier.len())
4. cache.prune_index()
5. Update last_maintenance timestamp

fn maybe_promote_on_access(&self, chunk_id, project_id) -> Option(TierDecision):
- If auto_promote_on_project_match and project_id matches chunk's project:
  - Get promotion score
  - If score >= promotion_threshold: promote immediately
  - Return decision
- Otherwise: return None (handled by periodic check_promotions)
