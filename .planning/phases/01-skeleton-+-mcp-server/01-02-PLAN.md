2. `crates/memd/src/mcp/protocol.rs`:
   Define JSON-RPC 2.0 message types per MCP spec:

   ```rust
   // JSON-RPC request
   pub struct Request {
       pub jsonrpc: String,  // Must be "2.0"
       pub id: Option<RequestId>,  // None for notifications
       pub method: String,
       pub params: Option<serde_json::Value>,
   }

   pub enum RequestId {
       Number(i64),
       String(String),
   }

   // JSON-RPC response
   pub struct Response {
       pub jsonrpc: String,
       pub id: Option<RequestId>,
       pub result: Option<serde_json::Value>,
       pub error: Option<RpcError>,
   }

   // JSON-RPC error object
   pub struct RpcError {
       pub code: i32,
       pub message: String,
       pub data: Option<serde_json::Value>,
   }
   ```

   Standard error codes:
   - PARSE_ERROR = -32700
   - INVALID_REQUEST = -32600
   - METHOD_NOT_FOUND = -32601
   - INVALID_PARAMS = -32602
   - INTERNAL_ERROR = -32603

   Implement:
   - Request::parse(line: &str) -> Result<Request>
   - Response::success(id, result) -> Response
   - Response::error(id, code, message) -> Response

3. `crates/memd/src/mcp/error.rs`:
   - McpError enum mapping to JSON-RPC error codes
   - Implement Into<RpcError> for McpError
   - Error variants for each MCP-specific failure mode

Create MCP server implementation (MCP-01):

1. `crates/memd/src/mcp/server.rs`:

   ```rust
   pub struct McpServer {
       config: Config,
       // Will hold store reference in next plan
   }

   impl McpServer {
       pub fn new(config: Config) -> Self;

       // Main server loop - reads from stdin, writes to stdout
       pub async fn run(&mut self) -> Result<()>;

       // Handle a single request
       async fn handle_request(&mut self, request: Request) -> Response;

       // MCP protocol handlers
       async fn handle_initialize(&self, params: Value) -> Result<Value>;
       async fn handle_tools_list(&self) -> Result<Value>;
       async fn handle_tools_call(&mut self, params: Value) -> Result<Value>;
   }
   ```

   Server loop:
   - Read lines from stdin (BufReader)
   - Parse as JSON-RPC Request
   - Dispatch to handler based on method:
     - "initialize" -> handle_initialize
     - "tools/list" -> handle_tools_list
     - "tools/call" -> handle_tools_call
     - Unknown -> METHOD_NOT_FOUND error
   - Write Response as JSON line to stdout
   - Flush after each response

   Initialize response:
   ```json
   {
     "protocolVersion": "2024-11-05",
     "capabilities": {
       "tools": {}
     },
     "serverInfo": {
       "name": "memd",
       "version": "0.1.0"
     }
   }
   ```

2. Update `crates/memd/src/main.rs`:
   - Parse CLI args with clap:
     - `--config <path>` - config file path
     - `--mode <mcp|cli>` - run mode (default: mcp)
   - Load config
   - Initialize tracing subscriber (JSON format for MCP mode)
   - Create and run McpServer

3. Add helper function `pub async fn run_server(config: Config)` for clean API.
