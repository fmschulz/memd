  <action>
Create core domain types following implementation_v0.md section 5.1:

1. `crates/memd/src/types.rs`:
   - `TenantId` - newtype wrapper around String with validation (non-empty, alphanumeric + underscore)
   - `ProjectId` - newtype wrapper around Option<String>
   - `ChunkId` - newtype wrapper around Uuid (UUIDv7 for time-sortability)
   - `ChunkType` - enum: Code, Doc, Trace, Decision, Plan, Research, Message, Summary, Other
   - `ChunkStatus` - enum: Draft, Final, Error, Deleted
   - `Source` - struct with uri, repo, commit, path, tool_name, tool_call_id (all Option<String>)
   - `MemoryChunk` - struct with all fields from spec:
     - chunk_id: ChunkId
     - tenant_id: TenantId
     - project_id: ProjectId
     - agent_id: Option<String>
     - timestamp_created: i64 (unix millis)
     - timestamp_observed: Option<i64>
     - chunk_type: ChunkType
     - status: ChunkStatus
     - source: Source
     - text: String
     - tags: Vec<String>
     - hash: String (content hash for dedupe)
   - Derive Serialize, Deserialize, Debug, Clone for all types
   - Implement Display for ChunkType and ChunkStatus

2. `crates/memd/src/error.rs`:
   - Use thiserror for ergonomic error definitions
   - `MemdError` enum with variants:
     - ConfigError(String) - config loading failures
     - ValidationError(String) - invalid input
     - StorageError(String) - storage operations
     - ProtocolError(String) - MCP protocol errors
     - IoError(#[from] std::io::Error)
     - SerializationError(#[from] serde_json::Error)
   - Define `pub type Result<T> = std::result::Result<T, MemdError>;`
  </action>
