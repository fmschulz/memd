Output: WAL writer that appends records with CRC-32 checksums and fsync durability.

```rust
//! Write-Ahead Log module
//!
//! Provides durability for write operations before segment commit.
//! Format: Magic(4B) | Type(1B) | Length(4B) | CRC32(4B) | Payload(variable)

pub mod format;
pub mod writer;

pub use format::{WalRecord, WalRecordType, WAL_MAGIC};
pub use writer::WalWriter;
```

**crates/memd/src/store/wal/format.rs:**
- WAL_MAGIC = *b"MWAL" (4 bytes)
- HEADER_SIZE = 13 (magic 4 + type 1 + length 4 + crc 4)
- WalRecordType enum (repr(u8)): Add = 1, Delete = 2, Checkpoint = 3
- WalRecord struct:
  - record_type: WalRecordType
  - tenant_id: String
  - chunk_id: String
  - timestamp: i64
  - payload: Vec<u8> (serialized chunk for Add, empty for Delete/Checkpoint)
- Implement encode_to_bytes(&self) -> Vec<u8>:
  - Serialize payload with bincode
  - Compute CRC-32 of payload
  - Write: magic || type || length || crc || payload
- Implement decode_from_bytes(bytes: &[u8]) -> Result<(WalRecord, usize)>:
  - Read header, validate magic
  - Read payload, verify CRC
  - Deserialize with bincode
  - Return record and bytes consumed
- Unit tests:
  - roundtrip_add_record
  - roundtrip_delete_record
  - roundtrip_checkpoint_record
  - detect_corrupt_crc

Implement the WAL writer with fsync durability.

**crates/memd/src/store/wal/writer.rs:**
- WalWriter struct:
  - file: File
  - path: PathBuf
  - records_written: u64
- WalWriter::create(path: &Path) -> Result<Self>
  - Create parent directories if needed
  - Open file with create + append mode
- WalWriter::open(path: &Path) -> Result<Self>
  - Open existing WAL for append
- WalWriter::append(&mut self, record: &WalRecord) -> Result<()>
  - Encode record to bytes
  - Write to file
  - Call file.sync_all() for durability (CRITICAL)
  - Increment records_written
- WalWriter::append_add(&mut self, tenant_id, chunk_id, timestamp, payload) -> Result<()>
  - Convenience method for Add records
- WalWriter::append_delete(&mut self, tenant_id, chunk_id, timestamp) -> Result<()>
  - Convenience method for Delete records
- WalWriter::append_checkpoint(&mut self, tenant_id, timestamp) -> Result<()>
  - Convenience method for Checkpoint records
- WalWriter::sync(&self) -> Result<()>
  - Explicit sync (already done per-record, but useful for batches)
- WalWriter::records_written(&self) -> u64
  - Return count for stats
- Unit tests:
  - write_single_record: create, append, verify file exists and is non-empty
  - write_multiple_records: append 3 different types, verify size grows
  - append_convenience_methods: test add/delete/checkpoint helpers

Note on durability: sync_all() is called after EVERY write. This is intentional - WAL durability is critical. Performance optimization (batching) comes later.
