# Phase 4.1: Pooling Strategy Support - Research

**Researched:** 2026-01-30
**Domain:** Embedding pooling strategies for sentence embeddings (mean vs last-token)
**Confidence:** HIGH

## Summary

This research investigates how to add pooling strategy support to memd's embedding pipeline to enable use of next-generation embedding models like Qwen3-Embedding-0.6B. The current implementation hardcodes mean pooling, which works for all-MiniLM-L6-v2 but is incompatible with decoder-style models that require last-token pooling.

The standard approach is to add a `PoolingStrategy` enum with `Mean` and `LastToken` variants, then dispatch in `run_inference()` based on model configuration. Last-token pooling extracts the hidden state at the final attended token position (the EOS token for decoder models). This requires attention mask handling to correctly identify the last non-padding token position.

The implementation is straightforward (2-4 hours) and follows established patterns from sentence-transformers and Qwen's official documentation. The main complexity is handling variable-length batches where padding direction matters.

**Primary recommendation:** Add `PoolingStrategy` enum to `EmbeddingConfig`, implement `last_token_pooling()` method using attention mask to find last attended position, and dispatch based on strategy in `run_inference()`.

**When to use:** When different models require different pooling approaches

### Pattern 2: Last-Token Pooling Implementation

**What:** Extract hidden state at last attended token position
**When to use:** Decoder-style models (Qwen3, E5-Mistral, GPT-based embedders)
**Example:**
```rust
// In onnx.rs
/// Last-token pooling: extract embedding at final attended position
///
/// For decoder models, the EOS token aggregates sequence information.
/// Must handle both left-padding and right-padding scenarios.
fn last_token_pooling(
    &self,
    hidden_states: &ArrayD<f32>,  // [batch, seq_len, hidden_size]
    attention_mask: &Array2<i64>, // [batch, seq_len]
) -> Array2<f32> {
    let batch_size = hidden_states.shape()[0];
    let hidden_size = hidden_states.shape()[2];

    let mut embeddings = Array2::<f32>::zeros((batch_size, hidden_size));

    for b in 0..batch_size {
        // Find last attended token (rightmost 1 in attention mask)
        let seq_len = attention_mask.shape()[1];
        let mut last_pos = seq_len - 1;

        // Scan from right to find last non-padding position
        // (handles right-padding; left-padding uses last position)
        for s in (0..seq_len).rev() {
            if attention_mask[[b, s]] == 1 {
                last_pos = s;
                break;
            }
        }

        // Extract hidden state at last position
        for h in 0..hidden_size {
            embeddings[[b, h]] = hidden_states[[b, last_pos, h]];
        }
    }

    embeddings
}
```

### Pattern 3: Model-Specific Configuration

**What:** Tie pooling strategy to model selection, not user config
**When to use:** When pooling strategy is inherent to model architecture
**Example:**
```rust
// In download.rs or a new model_config.rs
pub enum EmbeddingModel {
    AllMiniLmL6V2,  // 384-dim, mean pooling
    Qwen3_0_6B,     // 1024-dim, last-token pooling
}

impl EmbeddingModel {
    pub fn dimension(&self) -> usize {
        match self {
            Self::AllMiniLmL6V2 => 384,
            Self::Qwen3_0_6B => 1024,
        }
    }

    pub fn pooling_strategy(&self) -> PoolingStrategy {
        match self {
            Self::AllMiniLmL6V2 => PoolingStrategy::Mean,
            Self::Qwen3_0_6B => PoolingStrategy::LastToken,
        }
    }

    pub fn model_url(&self) -> &'static str {
        match self {
            Self::AllMiniLmL6V2 =>
                "https://huggingface.co/Xenova/all-MiniLM-L6-v2/resolve/main/onnx/model_quantized.onnx",
            Self::Qwen3_0_6B =>
                "https://huggingface.co/onnx-community/Qwen3-Embedding-0.6B-ONNX/resolve/main/onnx/model_q8.onnx",
        }
    }
}
```

### Anti-Patterns to Avoid
- **Hardcoding dimension:** Don't assume 384 everywhere; derive from model config
- **Ignoring attention mask:** Last-token pooling MUST use attention mask to handle padding
- **User-configurable pooling:** Pooling is model-specific, not a user choice
- **Index without rebuild detection:** Dimension mismatch must error clearly, not silently fail

## Don't Hand-Roll

Problems that look simple but have existing solutions:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Token position finding | Complex gather ops | Simple loop with attention mask | ndarray doesn't have PyTorch's gather; loops are clear |
| Model config management | Manual constants | Enum-based dispatch | Type safety, prevents mismatched pooling/dimension |
| Padding direction handling | Assume left or right | Check attention mask pattern | Models vary; both exist in the wild |

**Key insight:** The implementation is actually simpler in Rust than complex tensor operations would suggest. A loop over batch items with attention mask checking is clearer and more maintainable than trying to replicate PyTorch's advanced indexing.

## Common Pitfalls

### Pitfall 1: Wrong Padding Direction Assumption

**What goes wrong:** Code assumes right-padding but model uses left-padding (or vice versa)
**Why it happens:** Different tokenizers have different padding_side defaults
**How to avoid:** Always use attention mask to find actual token positions, never assume
**Warning signs:** Embeddings are all zeros or identical across different inputs

### Pitfall 2: Dimension Mismatch Silent Failure

**What goes wrong:** Existing 384-dim index receives 1024-dim embeddings; search fails silently or panics
**Why it happens:** Model changed but index wasn't rebuilt
**How to avoid:** Check dimension at insert/search time, fail with clear error and rebuild instructions
**Warning signs:** Search returns no results, or panics on vector operation

### Pitfall 3: Missing EOS Token

**What goes wrong:** Last-token pooling returns wrong position; embeddings are garbage
**Why it happens:** Tokenizer not configured to add EOS token for decoder models
**How to avoid:** Verify tokenizer config; some models (E5-Mistral) require explicit `add_eos_token: true`
**Warning signs:** Embeddings don't match official model outputs

### Pitfall 4: Not Normalizing After Pooling

**What goes wrong:** Cosine similarity scores are not in [0,1] range
**Why it happens:** Normalization step skipped or applied at wrong stage
**How to avoid:** Always L2 normalize final embeddings before storage/comparison
**Warning signs:** Similarity scores > 1.0 or < -1.0

### Pitfall 5: Instruction Formatting for Qwen3

**What goes wrong:** Query embeddings don't match expected quality; recall drops
**Why it happens:** Qwen3 expects instruction prefix for queries but not for documents
**How to avoid:** Use `Instruct: {task}\nQuery:{text}` format for queries only
**Warning signs:** Semantic search underperforms expectations (+1-5% improvement expected with instructions)

## Code Examples

Verified patterns from official sources:

### Last-Token Pooling (from Qwen3 docs)
```rust
// Reference: https://huggingface.co/Qwen/Qwen3-Embedding-0.6B
// Rust adaptation of official Python implementation

fn last_token_pool(
    last_hidden_states: &ArrayD<f32>,  // [batch, seq_len, hidden]
    attention_mask: &Array2<i64>,       // [batch, seq_len]
) -> Array2<f32> {
    let batch_size = last_hidden_states.shape()[0];
    let seq_len = last_hidden_states.shape()[1];
    let hidden_size = last_hidden_states.shape()[2];

    // Check for left-padding: if all batch items have mask=1 at last position
    let mut all_attended_at_end = true;
    for b in 0..batch_size {
        if attention_mask[[b, seq_len - 1]] != 1 {
            all_attended_at_end = false;
            break;
        }
    }

    let mut embeddings = Array2::<f32>::zeros((batch_size, hidden_size));

    if all_attended_at_end {
        // Left-padding case: last position is always the token we want
        for b in 0..batch_size {
            for h in 0..hidden_size {
                embeddings[[b, h]] = last_hidden_states[[b, seq_len - 1, h]];
            }
        }
    } else {
        // Right-padding case: find last attended position per batch item
        for b in 0..batch_size {
            // Sum attention mask to get sequence length (minus 1 for index)
            let seq_length: i64 = attention_mask.row(b).iter().sum();
            let last_pos = (seq_length - 1).max(0) as usize;

            for h in 0..hidden_size {
                embeddings[[b, h]] = last_hidden_states[[b, last_pos, h]];
            }
        }
    }

    embeddings
}
```

### Pooling Dispatch in run_inference
```rust
// In onnx.rs run_inference()

// After getting output_array from ONNX session:
let embeddings = match self.config.pooling {
    PoolingStrategy::Mean => {
        self.mean_pooling(&output_array, &attention_mask)
    }
    PoolingStrategy::LastToken => {
        self.last_token_pooling(&output_array, &attention_mask)
    }

// Normalize if configured (applies to both strategies)
if self.config.normalize {
    self.normalize(&mut embeddings);
}
```

## State of the Art

| Old Approach | Current Approach | When Changed | Impact |
|--------------|------------------|--------------|--------|
| Mean pooling for all | Strategy-based pooling | 2024-2025 | Enables decoder-style models (Qwen3, E5-Mistral) |

**Deprecated/outdated:**
- **CLS token pooling:** Only for encoder-only models; decoders use last-token
- **Fixed 384/768 dimensions:** Modern models support MRL (variable dims 32-1024)
- **Single pooling assumption:** Must support multiple strategies for model flexibility
