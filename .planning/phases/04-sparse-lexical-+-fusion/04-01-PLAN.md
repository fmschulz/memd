Create crates/memd/src/text/ module with:

**text/mod.rs:**
- Export TextProcessor, CodeTokenizer, SentenceSplitter
- Re-export key types

**text/tokenizer.rs - CodeTokenizer:**
- Implement code-aware tokenization:
  - Split camelCase: "parseUserInput" -> ["parse", "User", "Input"]
  - Split snake_case: "parse_user_input" -> ["parse", "user", "input"]
  - Preserve operators and keywords for exact matching
  - Detect and preserve acronyms (2+ consecutive uppercase: HTTP, API, SQL)
- Implement prose tokenization:
  - Lowercase non-acronym terms
  - Apply stemming via rust-stemmers (Porter or Snowball English)
- Create TokenType enum: Code, Prose, Mixed
- Implement tantivy Tokenizer trait for integration

**text/sentence.rs - SentenceSplitter:**
- Split text into sentences using unicode-segmentation
- Handle code blocks specially (preserve as single "sentence")
- Detect code vs prose heuristically:
  - Code indicators: braces, semicolons, function keywords
  - Multiple consecutive lines with code patterns -> code block
- Return Vec<Sentence> where Sentence has text, offset, is_code flag

**text/mod.rs - TextProcessor:**
- Combine tokenizer + sentence splitter
- process_chunk(text) -> Vec<ProcessedSentence>
- ProcessedSentence: { text, tokens, is_code, offset }

Update crates/memd/src/lib.rs to export text module.
