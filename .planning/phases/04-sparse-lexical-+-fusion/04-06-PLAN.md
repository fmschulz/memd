**Suite structure (following retrieval.rs pattern):**

```rust
//! Hybrid retrieval evaluation (Suite C)
//!
//! Compares hybrid (dense+sparse) against dense-only.
//! Measures quality improvement and performance baseline.

/// Quality metrics per query type
pub struct TypeMetrics {
    pub recall_at_10: f64,
    pub mrr: f64,
    pub precision_at_10: f64,
    pub query_count: usize,
}

/// Performance metrics
pub struct PerformanceMetrics {
    pub p50_ms: f64,
    pub p90_ms: f64,
    pub p99_ms: f64,
    pub mean_ms: f64,
    pub query_count: usize,
}

/// Run hybrid evaluation tests
pub fn run_hybrid_tests(memd_path: &PathBuf) -> Vec<TestResult>;
```

**Test cases:**

**C1_index_documents:**
- Load hybrid_test.json
- Index all documents
- Verify all indexed successfully

**C2_keyword_query_quality:**
- Run keyword queries only
- Measure Recall@10, MRR, Precision@10
- Compare against threshold: Recall > 0.9 for keyword queries
- Print metrics breakdown

**C3_semantic_query_quality:**
- Run semantic queries only
- Measure same metrics
- Threshold: Recall > 0.7 (semantic is harder)

**C4_mixed_query_quality:**
- Run mixed queries
- Measure metrics
- Should be between keyword and semantic performance

**C5_hybrid_vs_dense_improvement:**
- Run ALL queries twice:
  1. With hybrid enabled (default)
  2. With dense-only (if possible via config, otherwise skip)
- Compare metrics:
  - Keyword queries: hybrid should improve significantly
  - Semantic queries: should be similar
- Print improvement percentages

**C6_performance_baseline:**
- Run 100 queries (repeated from dataset)
- Measure latencies
- Calculate percentiles: p50, p90, p99
- Print performance baseline
- Target: p50 < 100ms, p99 < 500ms (adjustable)

**C7_quality_thresholds:**
- Check combined metrics against thresholds:
  - Overall Recall@10 > 0.75
  - Overall MRR > 0.6
  - Keyword Recall > 0.85
- PASS if all met, FAIL otherwise
