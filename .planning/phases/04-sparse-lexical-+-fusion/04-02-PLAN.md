```rust
//! Sparse index trait for keyword-based retrieval

/// Result from sparse search
pub struct SparseSearchResult {
    pub chunk_id: ChunkId,
    pub score: f32,  // BM25 score
    pub sentence_idx: usize,  // Which sentence matched
}

/// Trait for sparse/lexical indexes
pub trait SparseIndex: Send + Sync {
    /// Index a chunk's sentences
    fn insert(&self, tenant_id: &TenantId, chunk_id: &ChunkId, sentences: &[String]) -> Result<()>;

    /// Search for matching chunks
    fn search(&self, tenant_id: &TenantId, query: &str, k: usize) -> Result<Vec<SparseSearchResult>>;

    /// Remove chunk from index
    fn delete(&self, tenant_id: &TenantId, chunk_id: &ChunkId) -> Result<bool>;

    /// Number of indexed documents for tenant
    fn doc_count(&self, tenant_id: &TenantId) -> Result<u64>;
}
```

- Fields: tenant_id (STRING stored), chunk_id (STRING stored), sentence_idx (U64 stored), text (TEXT indexed)
- Use CodeTokenizer from text module for the text field

**Bm25Index struct:**
```rust
pub struct Bm25Index {
    index: tantivy::Index,
    reader: tantivy::IndexReader,
    writer: Mutex<tantivy::IndexWriter>,
    schema: Schema,
    // Field accessors
    tenant_field: Field,
    chunk_field: Field,
    sentence_field: Field,
    text_field: Field,
}
```

**Implementation:**

1. `new(path: Option<PathBuf>)` - Create in-memory or persistent index
   - Build schema with appropriate field options
   - Register CodeTokenizer as custom tokenizer
   - Create index writer with reasonable memory budget (50MB)

2. `insert()`:
   - For each sentence, create a document with tenant_id, chunk_id, sentence_idx, text
   - Add to writer, commit after batch

3. `search()`:
   - Parse query using QueryParser for text field
   - Add tenant_id filter using BooleanQuery
   - Execute search, collect top K
   - Return SparseSearchResult with chunk_id, score, sentence_idx

4. `delete()`:
   - Build term query for chunk_id field
   - Add tenant_id constraint
   - Delete matching docs, commit

5. `doc_count()`:
   - Query count of docs matching tenant_id

**Thread safety:**
- IndexReader is cloneable and thread-safe for reads
- IndexWriter wrapped in Mutex for writes
- Periodic commit (every N inserts or time-based)

**Tokenizer registration:**
Register CodeTokenizer with tantivy's tokenizer manager so queries use same tokenization as indexing.
